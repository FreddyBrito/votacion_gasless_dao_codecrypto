# DAO Gasless Voting — Frontend

Next.js 15 frontend for the DAO Gasless Voting system. Connects to MetaMask, displays proposals, and enables gasless voting via EIP-712 meta-transactions.

## Stack

- Next.js 15 (App Router, Turbopack)
- TypeScript
- Tailwind CSS v4
- ethers.js v6

## Setup

```bash
npm install
```

Configure `web/.env.local` (auto-generated by `start.sh`):

```
NEXT_PUBLIC_DAO_ADDRESS=0x...
NEXT_PUBLIC_FORWARDER_ADDRESS=0x...
NEXT_PUBLIC_CHAIN_ID=31337
NEXT_PUBLIC_RPC_URL=http://127.0.0.1:8545
RELAYER_PRIVATE_KEY=0x...
RELAYER_ADDRESS=0x...
```

## Development

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Build

```bash
npm run build
```

## Components

| Component       | Description                              |
|-----------------|------------------------------------------|
| `ConnectWallet` | Connects MetaMask, shows address + balance, handles chain switching |
| `FundingPanel`  | Deposit ETH to DAO, displays user/total balances |
| `CreateProposal`| Form with recipient, amount, deadline; validates 10% balance requirement |
| `ProposalList`  | Fetches all proposals from contract, displays in reverse order |
| `ProposalCard`  | Shows proposal details, vote counts, state badge, vote/execute buttons |
| `VoteButtons`   | 3 vote types (For/Against/Abstain), signs EIP-712 and sends to relayer |

## API Routes

### POST `/api/relay`

Receives a signed meta-transaction and submits it via the MinimalForwarder.

**Request body:**
```json
{
  "request": { "from": "0x...", "to": "0x...", "value": 0, "gas": 500000, "deadline": 1234567890, "data": "0x..." },
  "signature": "0x..."
}
```

**Response:**
```json
{ "txHash": "0x...", "success": true, "blockNumber": 1234 }
```

### POST `/api/execute`

Scans all proposals and executes eligible ones (past deadline, votes For > Against).

**Response:**
```json
{ "scanned": 5, "executed": [1, 3], "failed": [] }
```

## Voting Flow

1. User clicks a vote button
2. `lib/eip712.ts` builds EIP-712 typed data with the vote
3. MetaMask prompts user to sign (no gas)
4. Signed request is sent to `/api/relay`
5. Relayer submits transaction via MinimalForwarder
6. DAOVoting records the vote using the original user's address

## Key Files

- **`lib/web3.tsx`** — `Web3Provider` context wrapping the app. Manages wallet connection, contract instances, and balance refresh.
- **`lib/eip712.ts`** — `buildAndSignVote()` creates and signs the forward request; `submitToRelayer()` sends it to the API.
- **`lib/contracts.ts`** — ABIs for MinimalForwarder and DAOVoting, plus TypeScript enums for `VoteType` and `ProposalState`.
